# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in any F² AI product, **please report it responsibly**.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: **security@fsquared.ai**

Include:
- Description of the vulnerability
- Steps to reproduce
- Affected product(s) and version(s)
- Potential impact

### Response Timeline

| Action | Timeframe |
|--------|-----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix or mitigation | Depends on severity |
| Public disclosure | After fix is deployed |

### Scope

This policy covers all F² AI products:
- [virusfor.ai](https://virusfor.ai)
- [onlyallow.ai](https://onlyallow.ai)
- [batchforge.ai](https://batchforge.ai)
- [batchdrone.com](https://batchdrone.com)
- [quirkbot.ai](https://quirkbot.ai)
- [marinecitizen.com](https://marinecitizen.com)

### Encryption

All production systems use TLS 1.3. Sensitive data is encrypted with AES-256-GCM.

## Supported Versions

We support security patches for the current production release of each product.
